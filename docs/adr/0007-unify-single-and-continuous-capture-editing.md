# 单段与连续截图共用一个编辑工作台入口

快速结果只保留一个 `编辑工作台` 入口。单段截图和通过 `继续截取下一段` 明确相连的多段截图，都会进入同一个工作台；多段内容自动按续截关系加载。用户不再需要先进入编辑页，再寻找分节合并并进入第二个工作台。

连续关系只由截图数据中的 `parentCaptureId` 建立，不再用来源 URL 猜测。同一网址在不同时间产生的独立截图不会被自动合并。

自动关系只负责默认路径，不替代手动能力。工作台保留折叠的 `添加其他截图` / `添加或排除截图`，用来兼容旧版缓存、从当前位置手动截出的分段和需要自行组合的 PPT 截图；这些截图仍可勾选、排除和排序，但不会在快速结果里形成第二个工作台。

`回到原文` 始终保留为独立动作；只要保存了截图终点，就额外提供 `查看结束位置`，截图确实存在剩余内容时再显示 `继续截取下一段`。续截时如果原标签页已经关闭，扩展会重新打开原 URL，等待页面内容达到保存位置后再开始下一段。

这样保留了单段截图的简单流程，也复用了多段模式的按需高清预览和流式导出能力。多段模式只让当前分段保留高清画布，已访问的其他分段在高清预览生成过程中同步保留百万像素以内的低清对照画布，切换时不再同步压缩整张大图。代价是内部仍保留两种渲染页面，但它们对用户都属于同一个产品概念和同一个入口。
