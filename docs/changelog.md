# Changelog

## 1.2.1（2026-07-16）

- 将扩展安装名称、中英文语言包、Chrome Web Store 标题、GitHub 文档、隐私政策和产品界面统一为唯一正式品牌名 `ScrollCatch · 收`。
- `Full Page Capture` 保留为功能说明与搜索语义，不再作为正式产品名的一部分。
- 增加 Windows Smart App Control 发布边界说明，并在打包时拒绝 EXE、MSI、DLL 和启动脚本误混入扩展 ZIP。

## 1.2.0（2026-07-15）

- 按最终视觉方向板完成 QIDU 品牌升级：使用暖白、琥珀金与炭黑视觉体系，并在弹窗、快速结果和编辑工作台统一展示 `ScrollCatch · 收`。
- 全套 16、32、48、128 像素扩展图标由同一份 512 像素卷轴母版生成，提升浏览器工具栏中的辨识度。
- 加入品牌文案 `收其全貌，存其有度。 / Capture the whole, preserve with restraint.` 与 `A QIDU Utility`。
- 本地项目目录由 `XF-FullPage-Capture` 更名为 `ScrollCatch`，GitHub 仓库同步更名为 `Yifo98/ScrollCatch`，README、隐私政策、商店文案和发布脚本链接全部同步。
- 新的语言、编辑草稿和页面生命周期状态使用 `scrollCatch:*` 命名空间；升级时仍能读取旧命名空间中的语言选择和编辑草稿，避免用户设置丢失。
- 保留长页续接、Esc 停止、框选范围、飞书 / Lark PPT 逐页捕获、自由裁切、自定义分页、多段排序、缓存管理和 PDF 导出能力。
- 默认界面保持 English，用户主动切换中文后继续在浏览器本机记忆选择。
- 版本升级为 `1.2.0`，用于 GitHub Release 与 Chrome Web Store 新包发布。

## 1.1.0（2026-07-15）

- 产品名称更新为 `ScrollCatch — Full Page Capture`；初步重名检索后避开已有的 `PageLoop` 与 `SnapWeave`。
- 使用统一的「Caught Sections」产品图标：三个内容分段与右侧捕获钩在浏览器工具栏中更醒目，并由同一份母版导出 16、32、48、128 像素资源。
- 弹窗、快速结果和编辑工作台新增中文 / English 切换，首次使用默认 English；语言入口带有可见标签，用户选择中文后会保存在浏览器本机并在下次打开时恢复。
- 统一编辑工作台保持纯蓝、截图内容优先的品牌视觉，移除四阶段导航，只保留 `自由裁切`、`自定义分页`、`导出` 三个核心工具组。
- 重做扩展弹窗：突出完整页面截图，收拢提示与快捷键，并统一品牌视觉。
- 截图完成后默认进入轻量快速结果；它只负责确认截图、续接长页和快速保存，裁切、分页、分节和缓存工具按需进入编辑工作台。
- 快速结果将 `进入编辑工作台` 放在左侧第一位，以静态浅蓝描边和轻阴影提示主操作，不再播放循环特效；`保存当前段 PDF` 放在第二位，PNG、JPEG 和分页图片收进 `更多导出格式`。
- 连续截图的快速结果新增全部相连分段的轻量预览，不进入工作台也能检查第一段至当前段的内容和衔接顺序。
- 长页未完成时显示整页完成比例和剩余高度，并将 `继续截取下一段` 提升为主操作；此时导出按钮明确标记为保存“当前段”。
- 单段编辑和连续多段整理收束为同一个 `编辑工作台` 入口；连续截图会自动进入多段模式，不再暴露第二个合并工作台。
- 编辑工作台在单段模式保留折叠的 `添加其他截图`，多段模式保留 `添加或排除截图`，兼容旧版缓存、手动分段和独立 PPT 截图的补选、排除与排序。
- 自由裁切支持拖动、八向缩放和重新拉出任意保留区域；移除 `排除此段` 与 `预览设置`，将含义不清的 `当前视图` 改为 `只留当前画面`，并补回明确的 `恢复全图`。
- 自由裁切进入工作台时默认可用；开关只控制裁切工具，不再把全文误裁成当前可见区，并会同步开启到全部分段。`恢复全图` 保持工具开启并恢复每段完整边界。
- 裁切区常驻显示当前裁切状态；`精确数值` 面板固定在裁切工具下方，打开时同步当前 X、Y、宽、高，输入后立即反馈到截图裁切框。
- 取消额外的 `分页设置` 面板；分页状态与页数直接显示在工具标题旁。分页图片和纸张方向收进页头 `导出设置`，缓存大小、刷新和清理操作移动到左侧常驻的 `截图缓存` 卡片。
- 快速结果不再为未编辑的截图写入空白草稿，避免覆盖工作台“自定义分页线默认开启”的初始化状态。
- 自定义分页线进入工作台时默认开启；分页状态提示与分页线标签增大，`按纸张生成`、`新增分页线`、`删除选中线` 和 `清除分页线` 增加完成状态图标。
- 自定义分页区增加 `A / +` 新增、`Delete / Backspace` 删除的常驻快捷键提示；拖动分页线靠近预览上下边缘时会自动滚动画布，最后一页可移动到内容底部。
- 三组核心工具移动到截图预览上方，并压缩为「标题与状态 + 操作」两行布局；工作台只保留页头一个导出按钮，避免上下重复操作。
- 多段导出默认勾选 `同步全部设置`，会同步活动段的裁切比例和导出参数；自定义分页线会按各段自身裁切尺寸、纸张和方向重新生成，不再直接复制活动段的分页位置。
- 快速结果和编辑工作台默认勾选 `导出后自动删除本次截图缓存`；需要保留原始分段继续编辑时可在导出前取消。未完成长页或已关联多段只做当前段快速导出时会安全保留缓存，直到最终合并导出，避免破坏续截关系。
- 工作台直接说明未导出缓存的保留规则：缓存保存在浏览器本机，最多保留最近 12 次截图，并提供大小查看与手动清理入口。
- 导出并清理缓存后显示完成面板，明确提供 `返回原文`、`返回快速结果` 与 `关闭此页`，避免留在已经不能二次编辑的工作台。
- 将全屏等待层改为非阻塞进度卡，完整预览尚未拼接完成时即可查看已经处理的内容。
- 将交互预览预算从 2 亿像素降至 3200 万像素；PNG、JPEG 和高清分页导出都读取原始截图切片，不牺牲最终清晰度。
- 编辑工作台的多段模式只保留活动分节的高清画布，离开分节后立即释放大型画布；未访问分节仍可直接参与高清导出。
- 已访问的非活动分段保留约百万像素以内的低清对照画布，双列布局可检查相邻段衔接；低清画布在预览生成时同步绘制，切换时不再压缩整张高清画布。
- 已完整截到页面结尾时不再显示误导性的“继续获取下一段”操作；只有确实存在剩余内容时才提供续截入口。
- 快速结果始终保留独立的 `回到原文`；存在保存终点时额外显示 `查看结束位置`，未截完时再显示 `继续截取下一段`；原标签页关闭时会重开原网页、等待长内容就绪并恢复到保存的位置。
- 完整单段导出并自动删除原始切片后，仍保留轻量来源信息供 `回到原文` 使用；依赖截图切片的高清再导出和结束位置操作会明确禁用。
- 捕获开始时在页面全部 frame 注册 `Esc` 停止桥接，页面或嵌入内容获得焦点时都能结束当前捕获；结束后自动卸载监听，避免影响网页原有快捷键。
- 框选范围补充弹窗与页面双重 `Esc` 取消桥接，修复弹窗关闭后焦点变化导致的失效；选定起点后实时显示宽 × 高和半透明范围预览。
- 多段自动整理只接受明确的续截关系，不再按相同 URL 把独立历史截图误合并。
- 从快速结果定位到保存的结束位置后，再在弹窗选择 `从当前位置`，新截图会自动作为下一段加入原组合；关联信息同时写入会话缓存，扩展后台被浏览器挂起后仍能续接。
- 工作台的段号改为按当前组合顺序显示，不再沿用不同截图各自的原始 `第 1 段`；左侧提供明确的 `管理与排序`、`上移当前段` 与 `下移当前段`，合并导出严格按该顺序处理。
- 多段主操作统一显示为 `合并导出 PDF`，完成面板显示 `合并 PDF 已保存`；单段仍显示 `导出 PDF` / `PDF 已保存`。
- 导出后只清理本次实际导出的组合，其他历史截图不会被误删；工作台新增 `清理其他历史缓存`，并在界面中明确区分本次组合与历史缓存。
- 工作台预览区常驻提示低清预览属于正常现象，高清导出仍读取原始截图切片，不影响最终清晰度。
- 修正工作台窄栏的按钮换行：缓存清理改为两行全宽操作，分段上移/下移保持单行；顶栏语言选择器改为与相邻按钮等高的横向布局，并缩短英文标签。
- PPT / 演示稿的分页 PNG、分页 JPEG 保持在快速结果栏，保留旧版常用的按页图片出口。
- 更新扩展名称、图标和说明，回到“完整网页截图”这一核心承诺；GitHub 提供源码和安装包，Chrome Web Store 更新仍单独进行。

## 1.0.2

- 将 PPT 能力合并进 `捕获完整页面`，移除独立的 PPT 专用按钮。
- 新增 PPT / 演示文稿按页捕获策略，识别到打开后的飞书 PPT、Lark 演示文稿或类似预览页时，每个截图切片对应一页。
- PPT 按页捕获改为只点击精确页码缩略图，并在截图前校验当前页码；如果落到错误页，会停止当前按页流程，避免静默跳页。
- 截图前新增可见内容加载等待，遇到图片、骨架屏、加载指示器时会短暂等待，减少未加载空白被截进结果。
- 收紧飞书 `docx` 内嵌附件预览识别，避免普通文档页被误切到 PPT/PDF 预览 iframe；自定义起点/终点默认作用于当前页面正文。
- 捕获 PPT 时优先使用缩略图栏跳页；不可用时退回主预览容器的页面步长定位，但不再使用普通长网页滚动步长切片。
- 飞书 PDF.js / PPT 预览页按页捕获时，优先读取当前页完整画布作为页图，避免可视区域截图导致半页、偏移或右侧灰块。
- 按页捕获会裁切主幻灯片区域，并尽量隐藏底部浮动控制条。
- 按页捕获结果页默认启用幻灯片边界分页线，方便导出分页 PNG、分页 JPEG 或 PDF。
- 按页捕获的结果页和合并编辑器不再显示网页截图的 `PDF参考区`；自定义分页和导出都会优先使用 PPT 截图页边界。
- 普通网页截图缩短每屏等待和截屏间隔，并在弹窗提示长网页可先滚到底预加载图片缓存。
- PPT 页码跳转失败时改为多策略重试和中文错误提示，减少 Windows 端停在旧页导致的硬失败。
- 修复结果页 `回到结束位置` 和 `从结束位置继续获取` 可能回到页面开头的问题；回跳时会优先使用当前段真实结束坐标，并按原截图目标选择可滚动容器。
- 优化弹窗命名、文案和配色，将入口改为 `网页捕获工作台`，按钮改为 `捕获完整页面`、`从当前位置捕获` 和 `框选捕获范围`。
- 结果页和合并编辑器将纸张、方向和导出清晰度收进高级 PDF 参数，补齐预览缩放、适配宽度和导出后自动清理缓存入口。
- 将飞书 / PPT 按页截图说明拆到 `docs/features/feishu-ppt-capture.md`，README 只保留入口摘要。
