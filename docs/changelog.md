# Changelog

## 1.0.2

- 将 PPT 能力合并进 `捕获完整页面`，移除独立的 PPT 专用按钮。
- 新增 PPT / 演示文稿按页捕获策略，识别到打开后的飞书 PPT、Lark 演示文稿或类似预览页时，每个截图切片对应一页。
- PPT 按页捕获改为只点击精确页码缩略图，并在截图前校验当前页码；如果落到错误页，会停止当前按页流程，避免静默跳页。
- 截图前新增可见内容加载等待，遇到图片、骨架屏、加载指示器时会短暂等待，减少未加载空白被截进结果。
- 收紧飞书 `docx` 内嵌附件预览识别，避免普通文档页被误切到 PPT/PDF 预览 iframe；自定义起点/终点默认作用于当前页面正文。
- 捕获 PPT 时优先使用缩略图栏跳页；不可用时退回主预览容器的页面步长定位，但不再使用普通长网页滚动步长切片。
- 飞书 PDF.js / PPT 预览页按页捕获时，优先读取当前页完整画布作为页图，避免可视区域截图导致半页、偏移或右侧灰块。
- 按页捕获会裁切主幻灯片区域，并尽量隐藏底部浮动控制条。
- 按页捕获结果页默认启用幻灯片边界分页线，方便导出分页 PNG、分页 JPEG 或 PDF。
- 按页捕获的结果页和合并编辑器不再显示网页截图的 `PDF参考区`；自定义分页和导出都会优先使用 PPT 截图页边界。
- 普通网页截图缩短每屏等待和截屏间隔，并在弹窗提示长网页可先滚到底预加载图片缓存。
- PPT 页码跳转失败时改为多策略重试和中文错误提示，减少 Windows 端停在旧页导致的硬失败。
- 修复结果页 `回到结束位置` 和 `从结束位置继续获取` 可能回到页面开头的问题；回跳时会优先使用当前段真实结束坐标，并按原截图目标选择可滚动容器。
- 优化弹窗命名、文案和配色，将入口改为 `网页捕获工作台`，按钮改为 `捕获完整页面`、`从当前位置捕获` 和 `框选捕获范围`。
- 结果页和合并编辑器将纸张、方向和导出清晰度收进高级 PDF 参数，补齐预览缩放、适配宽度和导出后自动清理缓存入口。
- 将飞书 / PPT 按页截图说明拆到 `docs/features/feishu-ppt-capture.md`，README 只保留入口摘要。
