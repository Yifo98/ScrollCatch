# 截图速度与 PPT 页码跳转

## 长网页截图速度

长网页截图的耗时主要来自三件事：

- Chrome `captureVisibleTab` 有调用频率限制，扩展需要在截图之间保留间隔，避免触发浏览器配额错误。
- 每一屏截图前都要滚动页面、等待布局稳定，再截图。
- 图片懒加载、骨架屏和 loading 状态会让扩展多等一小段时间，避免把空白内容截进结果。

`1.0.2` 将普通网页的可见内容等待上限调低，并把截图间隔从 900ms 调到 700ms。遇到图片特别多的网页，建议先手动滚到页面底部，让图片和懒加载内容进入浏览器缓存，再回到要开始的位置点击扩展截图。

## PPT 页码跳转失败

Windows 或部分在线预览器里，页码输入框、PDF.js 当前页状态和左侧缩略图选中状态可能不同步。之前只要检测到当前页不是目标页，就会直接报错，例如：

```text
PPT page navigation landed on page 93 instead of 1.
```

`1.0.2` 改为多策略跳页：先尝试 PDF.js 页码跳转，再尝试真实页元素滚入视图，再尝试点击左侧缩略图，最后才用估算位置滚动。跳页后会等待页码和可见页同步；如果仍然失败，会显示中文提示，建议用户先点到目标页后重试，或使用 `从当前位置捕获`。
