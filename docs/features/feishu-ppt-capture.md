# 飞书 / PPT 按页截图

## 目标

`1.0.2` 的 `捕获完整页面` 会自动识别打开后的飞书 PPT、Lark 演示文稿和类似在线预览页。识别到 PPT / 演示文稿后，扩展不再按普通长网页的滚动步长切片，而是按页捕获。

这个功能解决两个问题：

- 普通滚动截图会把一页幻灯片截成上下两段，结果页看起来像页面错位。
- PPT 预览器里左侧缩略图、顶部栏、底部控制条和主幻灯片区域同时存在，普通滚动容器识别容易选错目标。

## 使用方式

1. 在飞书文档中先点击 PPT 附件，打开 PPT 预览页或独立查看页。
2. 点击扩展图标。
3. 选择 `捕获完整页面`。
4. 扩展会自动判断：PPT / 演示文稿按页捕获，普通网页按滚动拼接。
5. 结果页会默认按幻灯片边界生成分页线，方便导出分页 PNG、分页 JPEG 或 PDF。

如果已经打开到中间某一页，可以选择 `从当前位置捕获`，扩展会尽量从当前页开始按页捕获。

按页捕获的 PPT 不使用普通网页截图的 `PDF参考区`。结果页和合并编辑器里的自定义分页线默认来自截图时的一页页边界，导出分页 PNG、分页 JPEG 或 PDF 时也会按这些页边界输出。

## 捕获策略

PPT 按页捕获会优先使用这些线索：

- 页面中存在 `1/116` 这类页码/总页数信号。
- 页面必须同时呈现真正的 PPT / 演示查看器形态，例如左侧缩略图栏、大面积主幻灯片区域，或明确的 PPT / 幻灯片身份。
- 页面或节点带有 ppt、slide、presentation、viewer、preview、canvas、演示、幻灯片等特征。
- 左侧存在缩略图栏时，优先尝试点击对应页缩略图跳转。
- 缩略图跳转只认准确页码，不再用“第几个可见缩略图”兜底，避免滚到后半段时把第 10、11、12 页误跳到其他页。
- 每页截图前会读取页码指示器校验当前页；如果当前页和目标页不一致，会停止当前按页流程，而不是继续截错页。
- 没有可靠缩略图时，退回到主预览容器的页面步长定位，但仍按页边界裁切当前幻灯片。
- Windows 或部分预览器页码状态不同步时，会先重试 PDF.js 跳页、真实页元素滚入视图、缩略图点击和估算滚动；如果仍然失败，会给出中文提示，可先手动切到目标页后重试，或选择 `从当前位置捕获`。

捕获每一页时，扩展会优先读取 PDF.js / PPT 预览器里当前页的完整画布作为页图。这样即使浏览器视口只能看到半页，也不会把上下两页拼在同一张切片里；如果站点或浏览器阻止读取画布，才退回主幻灯片区域裁切和可视截图。截图前还会等待当前可见区域里的图片、加载指示器、骨架屏和常见懒加载状态短暂稳定，减少空白页或半加载内容被截进结果。

在飞书 `docx` 正文中，如果 PPT 只是一个附件预览卡片或 `mount_point=docx_file` 的小型预览 iframe，`捕获完整页面` 会优先按当前文档页滚动捕获，不会把这个卡片误认为已经打开的 PPT 查看器。

## 边界

- 如果 PPT 只是正文里的附件封面卡片，扩展不能自动展开所有幻灯片。必须先打开 PPT 预览页。
- `框选捕获范围` 面向当前页面正文，不会主动切到 PPT 按页识别；需要按页捕获 PPT 时，请先打开 PPT 预览页，再用 `捕获完整页面` 或 `从当前位置捕获`。
- 如果目标网站使用非常规虚拟渲染，且页面没有页码、缩略图或可识别的主幻灯片节点，扩展会退回普通滚动截图。
- 如果预览页位于跨域 iframe 中，扩展会尝试注入并选择 iframe 内的目标；如果浏览器或站点阻止注入，则会退回外层页面。
- 如果缩略图列表本身还没加载完成，扩展会等待准确页码出现；如果一直拿不到目标页缩略图，会退回页面定位或停止按页流程，避免用错误缩略图继续截图。

## 对应实现

- `popup/popup.html`：将 PPT 能力合并进 `捕获完整页面`，不再提供独立 PPT 按钮。
- `background.js`：识别 `captureStrategy: "pages"` 后走按页捕获循环。
- `content/capture-target.js`：识别 PPT 查看器、页数、缩略图栏和主幻灯片裁切区域。
- `result/result.js`：按页捕获结果默认启用幻灯片边界分页线。
- `result/merge.js`：合并编辑器按每段 PPT 自己的截图页边界分页，不再套用普通网页的 PDF 参考区。
